# ฐานข้อมูลระบบครุภัณฑ์องค์กรนักศึกษา

ไฟล์ `schema.sql` เป็น schema สำหรับ PostgreSQL และ Supabase พร้อมใช้งานจริง โดยรองรับปีงบประมาณตั้งแต่ `2561` เป็นต้นไป และเพิ่มปีใหม่ในอนาคตได้ เช่น `2568`, `2569`, `2570`

ไฟล์ `seed-organizations.sql` เป็นข้อมูลตั้งต้นสำหรับ Dropdown ฝ่าย/ชมรมที่รับผิดชอบ สามารถรันซ้ำได้โดยไม่สร้างชื่อซ้ำ เพราะใช้ `on conflict (organization_name) do update`

## ตารางหลัก

- `organizations` เก็บข้อมูลสโมสรนักศึกษา สภานักศึกษา ฝ่าย ชมรม และชมรมจังหวัด
- `users` เก็บผู้ใช้งานและบทบาท โดย `password` ต้องเป็นค่าที่ hash แล้ว
- `assets` เก็บข้อมูลครุภัณฑ์ทั้งหมด ผูกกับ `organizations`
- `asset_images` เก็บรูปภาพหลายรูปต่อครุภัณฑ์
- `annual_inspections` เก็บผลตรวจสอบประจำปี โดย 1 ครุภัณฑ์มีผลตรวจได้หลายปี
- `activity_logs` เก็บประวัติการใช้งานและการแก้ไขข้อมูล

## วิธีใช้งาน

นำเนื้อหาใน `schema.sql` ไปรันใน Supabase SQL Editor หรือรันกับ PostgreSQL ผ่านเครื่องมือจัดการฐานข้อมูลที่ใช้อยู่

ถ้าสร้างตารางไว้แล้วและต้องการเพิ่มเฉพาะรายชื่อฝ่าย/ชมรม ให้รัน `seed-organizations.sql`

โครงสร้างนี้มี foreign keys, unique constraints, check constraints, indexes และ trigger สำหรับ `updated_at` เรียบร้อยแล้ว
